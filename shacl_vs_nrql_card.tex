{\fontsize{26}{31}\selectfont\bfseries\color{ink}
What did \textcolor{shaclblue}{SHACL} (2017) add that\\[3pt]
RacerPro \textcolor{nrqlgreen}{nRQL} couldn't already do in 2005?\par}

\vspace{12pt}

%% ---------- Example 1: required property ----------
\begin{center}\begin{minipage}{0.82\textwidth}\begin{tcbraster}[raster columns=2, raster column skip=6mm, raster equal height=rows]
\begin{shaclbox}{SHACL Core --- required property}
ex:PersonShape
  a sh:NodeShape ;
  sh:targetClass ex:Person ;
  sh:property [
    sh:path ex:firstName ;
    sh:minCount 1 ] .
\end{shaclbox}
\begin{nrqlbox}{RacerPro nRQL (2005) --- missing known value}
(retrieve (?x)
  (and
    (?x Person)
    (neg
      (project-to (?x)
        (?x ?y firstName)))))
\end{nrqlbox}
\end{tcbraster}\end{minipage}\end{center}
{\Large\color{muted} A \textbf{negated projected subquery} $=$ \texttt{NOT EXISTS} / anti-join --- nRQL had it before SPARQL\,1.1 (2013) standardized \texttt{FILTER NOT EXISTS}.\par}

\vspace{11pt}

%% ---------- Example 2: value type (with DL entailment) ----------
\begin{center}\begin{minipage}{0.82\textwidth}\begin{tcbraster}[raster columns=2, raster column skip=6mm, raster equal height=rows]
\begin{shaclbox}{SHACL Core --- every value must be a Person}
ex:ParentShape
  a sh:NodeShape ;
  sh:targetClass ex:Parent ;
  sh:property [
    sh:path ex:hasChild ;
    sh:class ex:Person ] .
\end{shaclbox}
\begin{nrqlbox}{RacerPro nRQL (2005) --- with DL reasoning}
(retrieve (?x)
  (and
    (?x Parent)
    (?x ?y hasChild)
    (neg
      (?y Person))))
\end{nrqlbox}
\end{tcbraster}\end{minipage}\end{center}
{\Large\color{muted} Violation $=$ a known child that is \emph{not} a \texttt{Person}. And \texttt{(?y Person)} uses \textbf{DL entailment} --- an \emph{inferred} Person passes, no materialization needed.\par}
